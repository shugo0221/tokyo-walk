プロジェクト要件定義書：Tokyo Walk Randomizer
1. プロジェクト概要
サービス名: Tokyo Walk Randomizer（東京散歩ガチャ）

。

開発手法: AI駆動開発（Cursor / Next.js / TypeScript）

2. ターゲットユーザー
散歩が趣味だが、行き先がマンネリ化している東京都内在住者・観光客。

「暑すぎる」「雨が降りそう」といった天候不安で外出を迷っている人。

3. 機能要件
① 状況判定ロジック
季節判定: システム日付から自動的に「春・夏・秋・冬」を判定。

環境入力: ユーザーが現在の「気温」「天候（晴天・曇天・雨天）」を選択・調整可能。

時間選択: 散歩の許容時間（30分 / 60分 / 90分）を選択。

② コース抽出・表示
フィルタリング: 上記の「季節・気温・天候・時間」の4条件すべてに合致するコースをデータ群から抽出。

ランダム表示: 合致したコースの中から、ボタン一つで1件をランダムに選出。

詳細表示: エリア名、コース名、その場所の魅力（説明文）、所要時間をカード形式で表示。

4. データ構造（Schema）
lib/courses.ts で管理される各コースの定義：

id: ユニークな識別番号

name: コース名（例：目黒川の桜道）

duration: 所要時間（30 / 60 / 90）

area: エリア名（例：中目黒、浅草）

seasons: 推奨季節の配列

weatherStyles: 対応天候の配列

description: コースの見どころ解説文
